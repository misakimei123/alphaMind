## 2. 个人项目的最低独立性要求

如果 alphaMind 是个人项目，项目负责人、策略负责人、数据负责人和运维负责人可以由同一人承担，但必须满足：

- Backtest Qualified、Paper Ready、Paper Qualified 和 Live Canary 上线至少由另一名独立评审者复核；
- 资金所有人必须明确确认最大可接受绝对损失；
- 真实交易 key 的创建、权限和撤销必须有可复核记录；
- 实现者不能用自评替代阶段门禁；
- 无法安排独立评审时，对应阶段保持 `BLOCKED`。

## 3. 团队项目的职责分离

如果 alphaMind 是团队项目：

- 每个角色必须填写具体姓名或稳定标识；
- 风险审批人与策略负责人原则上分离；
- 运维负责人必须有替补；
- 告警升级路径必须包含时限和最终停机权；
- Live Canary 至少需要资金所有人、风险审批人和运维负责人共同确认。

## 4. 当前结论

个人项目的研究与实现阶段责任安排已经明确：

- Codex 是实施代理，不持有资金、密钥或上线批准权；
- 项目所有人负责范围、资金、风险、门禁和运维决策；
- Phase 3 前补充紧急联系人、告警渠道和响应时限；
- Live Canary 前补充外部 runbook 复核。

P0-01 已完成。项目级最大可接受损失采用 10% 比例与 45 USDT 固定上限中的更严格者；Codex 无资金和上线授权。
