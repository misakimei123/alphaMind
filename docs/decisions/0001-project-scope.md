## 0. 2026-07-18 范围修订

本 ADR 的个人项目、Bybit、Freqtrade 唯一交易写入者、API 无提现权限和项目所有人承担资金责任继续有效。以下旧范围已被 [完整开发计划](../development-plan.md) R0-R6 显式取代：

- “单一现货”改为 Bybit Spot + USDT Linear Perpetual；
- “BTC/ETH 固定”改为 Instrument Registry 配置，初始 BTC/ETH/SOL/HYPE；
- “long/flat、无杠杆”改为现货 long、合约按配置 long/short 和最大杠杆；
- “4h Donchian 唯一策略”改为默认 30 分钟 AI 决策周期，Donchian/ATR/EMA 作为特征和对照基线；
- “AI 不进入实时资金链”改为 AI 生成结构化 Action、Telegram 人工授权、确定性风控、Freqtrade 自动执行；
- P0-P5 旧晋升门禁改为 R0-R6 纵向闭环。

现有 `min(现金流调整资本基线 × 10%, 45 USDT)` 在 R0-05 前作为保守默认值保留；是否调整、Spot/Futures 资金分配、最大杠杆、short 和亏损加仓必须由项目所有人在 R0-05 明确确认。

本文第 1-7 节保留 2026-07-15 的历史决策背景；发生范围冲突时以上述修订和开发计划为准。

## 5. 损失和停止边界

根据当前已确认范围：

- 计划总资金：约 450–500 USDT；
- Live Canary 初始比例：5%–10%；
- 因此 Live Canary 初始分配的理论区间为 22.50–50.00 USDT；
- 实际金额必须在上线审批时从区间中选择一个固定值；
- 最大可接受损失：现金流调整资本基线的 10% 与固定 45 USDT，达到任一边界即停止；
- 该上限等于 450 USDT 的 10%，并低于 500 USDT 的 10%，因此在当前计划资金区间内不超过用户表达的 10% 边界。

有效项目级停止边界为 `min(现金流调整资本基线 × 10%, 45 USDT)`。它与 Live Canary 分配额、单笔计划风险、单日亏损、单周亏损和回撤阈值是不同层级的约束，后续必须同时满足。任何资金规模调整都不能自动提高固定上限；提高比例或固定上限必须由项目所有人重新明确确认并更新本决策。

## 6. 个人项目治理安排

| 事项 | 当前安排 |
|---|---|
| 项目、资金与最终风险责任 | 项目所有人（用户） |
| 设计与代码实施 | Codex 实施代理 |
| Codex 权限边界 | 无资金授权，不创建或持有真实交易 key，不批准 Live Canary |
| 阶段门禁确认 | 项目所有人（用户）复核 Codex 交付物后明确确认 |
| 生产运维责任 | 项目所有人（用户）；Phase 3 前必须补齐告警和紧急操作安排 |
| Live Canary 批准 | 项目所有人（用户）以版本化记录明确批准 |

该安排满足“实现者不能自行批准交付物”的最低要求：Codex 负责实现，项目所有人负责门禁与资金决策。进入 Live Canary 前仍应安排另一名可信人员复核密钥撤销和紧急停机 runbook；在此之前该项不阻塞研究和代码阶段。

## 7. 评审结论

当前结论：`DONE`。

依据：

- I-01 至 I-08 已全部确认；
- 最大可接受损失使用 10% 比例与 45 USDT 固定上限，达到任一边界即停止；
- 历史 v1 开发目标为 Bybit 现货 BTC/ETH；当前目标已扩展为配置化 Spot/Futures；常驻地区仍不进入策略代码路径；
- 个人项目中 Codex 负责实施，项目所有人保留资金、风险和阶段门禁权；
- P0-01 不授权 live trading、真实 key 或资金移动。

以上是 2026-07-15 的历史结论。当前下一任务和开发权限以 [完整开发计划](../development-plan.md) 为准。
